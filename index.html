로그인·동기화

직접 등록과 일괄등록

직접 등록

새로운 여행을 단계별로 만들고 필요한 내용을 하나씩 입력할 때 사용합니다.

일괄등록

과거 여행이나 일정이 많을 때 표 붙여넣기 또는 CSV 업로드로 한 번에 등록합니다.

개인 일정과 그룹 일정

개인 일정

내 여행 준비 데이터에 저장하는 교통 또는 숙소입니다. 개인 교통을 저장하는 것만으로 그룹 화면에 자동 공개되지는 않습니다.

그룹 일정

그룹코드로 같은 그룹을 불러와 함께 확인하는 교통 또는 숙소입니다. 개인 교통은 ‘현재 그룹에 공유’를 눌렀을 때 별도 그룹 일정으로 저장됩니다.

그룹 숙소는 참여자별 숙소를 복제하지 않고 하나의 그룹 숙소와 참여자 목록으로 저장됩니다. 같은 그룹코드를 연결하면 공유된 교통과 숙소를 불러오며, 그룹 숙소의 참여자는 각자 설정된 투숙 기간으로 표시됩니다. 그룹코드는 그룹 일정 조회·공유에 필요하고, 이메일 로그인은 같은 계정의 개인 준비 데이터를 기기 간 동기화할 때 사용합니다.

여행 데이터 가져오기 안내

  • 표 붙여넣기 또는 CSV 업로드를 지원합니다.
  • 한 행에 데이터 하나를 입력하고 참여자는 쉼표로 구분합니다.
  • 국가코드는 JP, HK, US 같은 2자리 코드를 사용합니다.
  • 통화는 JPY, HKD, USD 같은 3자리 코드를 사용합니다.
  • 여행, 페스티벌, 교통, 숙소, 짐싸기를 등록할 수 있습니다.
  • 기존 여행과 연결하거나 여행·페스티벌 행으로 새 여행을 만들 수 있습니다.

저장 및 동기화

로그인하지 않은 데이터는 현재 브라우저의 로컬 저장소에 저장됩니다. 이메일로 로그인하면 같은 계정으로 로그인한 기기에서 준비 데이터를 불러오고 저장할 수 있으며, ‘지금 동기화’는 서버와 현재 데이터를 즉시 맞춥니다.

그룹코드는 일행이 공유한 교통·숙소를 불러오는 용도이고 로그인 동기화와는 별개입니다. 데이터 초기화는 화면에 안내된 여행 일정, 숙소, 준비도, 짐싸기, 환전과 비용 데이터를 초기 상태로 되돌립니다. 로그인 중 초기화하면 그 상태도 계정 데이터에 동기화됩니다.

FAQ

처음에는 무엇부터 등록해야 하나요?

새 여행 만들기에서 여행명과 행사·여행 기간을 먼저 등록한 뒤 교통, 숙소와 준비물을 채워보세요.

직접 등록과 일괄등록 중 무엇을 사용해야 하나요?

한두 개 여행은 직접 등록이 간단합니다. 과거 기록이나 여러 일정이 있다면 표 또는 CSV 일괄등록이 빠릅니다.

개인 일정과 그룹 일정은 무엇이 다른가요?

개인 일정은 내 준비 데이터에 저장되고, 그룹 일정은 그룹코드로 연결된 공유 데이터입니다. 개인 교통은 공유 버튼을 눌러야 그룹에 별도로 등록됩니다.

로그인하지 않아도 데이터가 저장되나요?

네. 현재 브라우저의 로컬 저장소에 저장됩니다. 브라우저 데이터를 지우거나 다른 기기를 사용하면 자동으로 이어지지 않습니다.

다른 기기에서도 같은 데이터를 볼 수 있나요?

같은 이메일 계정으로 로그인하고 동기화하면 개인 준비 데이터를 다른 기기에서 불러올 수 있습니다. 그룹 일정은 해당 그룹코드도 연결해야 합니다.

일괄등록한 내용도 나중에 수정할 수 있나요?

등록 후 각 여행 화면의 기존 관리 기능을 사용합니다. 숙소와 준비물은 수정할 수 있고, 수정 버튼이 없는 교통 항목은 삭제 후 다시 등록해야 할 수 있습니다.

여행마다 다른 통화를 사용할 수 있나요?

일괄등록한 여행에는 ISO 통화코드가 함께 저장됩니다. 환전 화면에서는 현재 사용할 기준 통화를 선택해 저장하고 필요할 때 변경합니다.

짐싸기 완료 상태도 저장되나요?

네. 여행별 체크 상태가 현재 브라우저에 저장되고, 로그인한 경우 준비 데이터 동기화 대상에 포함됩니다.

문제가 있거나 의견이 있나요?

오류 신고와 기능 제안을 보내주시면 사이트 개선에 도움이 됩니다.

오류 화면과 발생 상황을 함께 보내주시면 확인에 도움이 됩니다.
